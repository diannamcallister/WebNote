JS Objects & Workflow

Select each section to learn about the JS objects created / helper functions called when performing a certain action through the UI.

When highlighting a text element that has an eventListener for the function nextToHighlight() from the WebNote library, the helper function createHighlightOptions() will be called, to create the markup options bar and display it on the page.

Below, you can take a look at the HTML and CSS that are used and formed in the JS file to create the markup options bar:

<div id="highlight">
    <div id="highlightOptions">
        <img class="dot" style="background-color: green;">
        <img class="dot" id="pink" style="background-color: pink;">
        <img class="dot" style="background-color: orange;">
        <img class="dot" style="background-color: yellow;">
        <img class="dot" src="https://webnote-lib.herokuapp.com/markup-bar-pics/no_highlight.png">
        <img class="dot" src="https://webnote-lib.herokuapp.com/markup-bar-pics/strikethrough.png">
        <img class="dot" src="https://webnote-lib.herokuapp.com/markup-bar-pics/bookmark.png">
        <img class="dot" src="https://webnote-lib.herokuapp.com/markup-bar-pics/notepad.png">

To create the outer box / rectangle surrounding the markup options, a div with the id highlight is added to the DOM, whose CSS style is as follows:
#highlight {
    border-style:solid;
    border-width:3px;
    padding-top: 5px;
    padding-bottom: 5px;
    margin:5px;
    height:80px;
    width:360px;
    overflow:hidden;
    text-align:center;
    background-color: white;
}
To make the markup options appear in circles, a class with the name dot is added to each of the elements of the markup options bar, whose CSS style is as follows:
.dot {
    height: 25px;
    width: 25px;
    border-radius: 50%;
    display: inline-block;
    margin-right: 10px;
    border: 1px solid black;
}

There are many options that can be chosen from the markup options bar.

Whenever one of the options are chosen, the helper functions clickHandler() and updateVisualOfText() are called to display the desired markup change to the user.

Take a look at the HTML and CSS performed when each markup option is selected:

When any of the green, pink, orange, or yellow circles are chosen from the markup bar, then the selected text that is desired to be highlighted will be added to a span element, and it will have its background color edited.
NOTE: due to the addition of the span element to display a highlighted text element, this library has decided that if text has already been highlighted, any further highlight modifications attempted (ex: changing the highlight to a different colour, highlighting content that overlaps with existing highlighted content, etc.) will be ignored. The user must first remove the highlight, and then is able to perform the different highlight modifications desired.
HTML changes:
# in this example, we are assuming that the highlight color chosen was yellow
<span style="background-color: yellow;">chosen text</span>
When the "no" symbol is selected, the removeHighlight() function is called to remove the span element that was previously added to the text (if the selected text was not highlighted, no action will be taken):
HTML before:
# in this example, we are assuming that the highlight color chosen was yellow
<p><span style="background-color: yellow;">chosen text</span></p>
HTML after:
<p>chosen text</p>
When the strikethrough option is chosen from the markup bar, then the selected text that is desired to be striked out will be added to a span element, and it will have its text decoration edited:

When the bookmark option is chosen from the markup bar, then the function createBookmark() is called to create a bookmark element (html element / div with id = bookmark) and add the bookmark element to the webpage, on the right hand side of the page at the relative height on the screen to the relative height of the chosen text compared to the height of the webpage (ex: if a bookmark is added for text at the bottom of a webpage, then the bookmark will displayed lower on the side of the screen).
HTML changes:
A div is added to the bottom of the body of the webpage, with the id bookmark and a style value for top depending on the height of the text on the webpage:
# in this example, we are assuming that the height of the text is 39.2931% of the webpage
<div id="bookmark" style="top: 39.2931%;"></div>
CSS changes:
A div with the id bookmark is added to the DOM when creating the bookmark, whose CSS style is as follows:
#bookmark {
    border-style:solid;
    border-width:2px;
    position: fixed;
    margin:1px;
    height:5px;
    width:20px;
    overflow:hidden;
    background-color: yellow;
    right: 0px;
    top: 50%;
}
When the note option is chosen from the markup bar, then a note element (html element / span with id = tooltip) is added around the selected text. Then, another span element (with id = tooltiptext) is added inside of the tooltip span, which will include the note information. When the note is added, a grey highlight and dotted underline is added to the text with the note, and when hovering over the text with the note, the note's contents is displayed.
HTML changes:
# in this example, we are assuming that highlighted text was "markers" and that the text of the added note was "test!"
<span class="tooltip" style="background-color: rgb(234, 234, 234);"> markers <span class="tooltiptext">test!</span></span>
CSS changes:
A span element with the id tooltip is added to the DOM when creating the note, whose CSS style is as follows:
.tooltip {
    position: relative;
    display: inline-block;
    border-bottom: 1px dotted black;
    opacity: 100;
    font: auto;
}
A span element with the id tooltiptext is added to the DOM when creating the note, whose CSS style is as follows:
.tooltip .tooltiptext {
    visibility: hidden;
    background-color: #eaeaea;
    border: 1px solid black;
    color: black;
    text-align: center;
    padding: 5px 5px 5px;
    border-radius: 6px;
    position: absolute;
    z-index: 1;
    width: auto;
    bottom: 100%;
    left: 50%;
}
When the tooltip span is hovered over, the note is shown. This is due to the CSS, whose style is as follows:
.tooltip:hover .tooltiptext {
    visibility: visible;
}
When selecting on the "X" option, the function removeHighlightOptions() is called and causes the markup options bar to be removed from the DOM. No other HTML modifications are made.
When selecting on the upload file text, the user is able to add a file to the DOM, which will replace the HTML of the current webpage through the function loadData(). No other HTML modifications are made.
When selecting on the download file text, the function saveData() is called, and the user is able to download an HTML file including all of the markup performed to the DOM. No other HTML modifications are made.
